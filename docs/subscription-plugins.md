# Subscription Plugin Development

Zoro uses a plugin-based subscription system that allows users to subscribe to paper feeds from various sources. This guide covers how the system works and how to create new source plugins.

## Overview

The subscription system lives in the `zoro-subscriptions` crate. At its core is the `SubscriptionSource` trait -- any Rust struct that implements this trait becomes a paper feed source.

The built-in source is **HuggingFace Daily Papers**, which fetches curated ML/AI papers from the HuggingFace community API. The architecture is designed to make adding new sources straightforward.

### How It Works

1. Each source plugin implements the `SubscriptionSource` trait
2. The desktop app runs a **background poller** that periodically calls `fetch()` on enabled subscriptions
3. Fetched items are stored as `SubscriptionItem` records in the database
4. Users can browse feed items in the UI and selectively add them to their library

## `SubscriptionSource` Trait -- Full API Reference

The trait is defined in `crates/zoro-subscriptions/src/source.rs`:

```rust
use async_trait::async_trait;
use zoro_core::models::SubscriptionItem;
use crate::error::SubscriptionError;

/// Trait that subscription source plugins implement
#[async_trait]
pub trait SubscriptionSource: Send + Sync {
    /// Unique identifier for this source type (e.g., "huggingface-daily")
    fn source_type(&self) -> &str;

    /// Human-readable display name
    fn display_name(&self) -> &str;

    /// Fetch new items, optionally since a given timestamp
    async fn fetch(
        &self,
        config: &serde_json::Value,
        since: Option<chrono::DateTime<chrono::Utc>>,
    ) -> Result<Vec<SubscriptionItem>, SubscriptionError>;

    /// Default configuration for this source
    fn default_config(&self) -> serde_json::Value;

    /// Description of what this source provides
    fn description(&self) -> &str;
}
```

### Method Details

| Method | Returns | Description |
|---|---|---|
| `source_type()` | `&str` | A unique, kebab-case identifier (e.g., `"huggingface-daily"`, `"arxiv-rss"`). Stored in the `subscriptions.source_type` database column. |
| `display_name()` | `&str` | A human-readable name shown in the UI (e.g., `"HuggingFace Daily Papers"`). |
| `fetch()` | `Result<Vec<SubscriptionItem>, SubscriptionError>` | The core method. Fetches items from the external source. Receives the user's config and an optional `since` timestamp for incremental fetching. |
| `default_config()` | `serde_json::Value` | Returns a JSON object with the default configuration for this source. Used when creating a new subscription. |
| `description()` | `&str` | A brief description of the source for the UI. |

### The `SubscriptionItem` Model

Each fetched item is represented as a `SubscriptionItem` (defined in `zoro-core`):

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SubscriptionItem {
    pub id: String,                     // UUID, generated by the plugin
    pub subscription_id: String,        // Set by the caller (leave empty in fetch)
    pub paper_id: Option<String>,       // None until added to library
    pub external_id: String,            // Source-specific ID (e.g., ArXiv ID)
    pub title: String,
    pub authors: Vec<Author>,
    pub abstract_text: Option<String>,
    pub url: Option<String>,
    pub pdf_url: Option<String>,
    pub html_url: Option<String>,
    pub upvotes: Option<i32>,
    pub data: Option<serde_json::Value>, // Source-specific extra data
    pub fetched_date: String,           // RFC 3339 timestamp
    pub added_to_library: bool,         // false when returned from fetch
}
```

### Error Handling

The `SubscriptionError` enum covers common failure modes:

```rust
#[derive(Error, Debug)]
pub enum SubscriptionError {
    #[error("HTTP request failed: {0}")]
    HttpError(#[from] reqwest::Error),

    #[error("Parse error: {0}")]
    ParseError(String),

    #[error("Source not found: {0}")]
    SourceNotFound(String),

    #[error("Subscription error: {0}")]
    Other(String),
}
```

Use `HttpError` for network failures (automatic via `#[from]`), `ParseError` for malformed responses, and `Other` as a catch-all.

## Step-by-Step: Creating a New Source Plugin

### 1. Create the Plugin Module

Add a new file in `crates/zoro-subscriptions/src/`, for example `arxiv_rss.rs`:

```rust
use async_trait::async_trait;
use zoro_core::models::{Author, SubscriptionItem};
use crate::error::SubscriptionError;
use crate::source::SubscriptionSource;

pub struct ArxivRssFeed {
    client: reqwest::Client,
}

impl ArxivRssFeed {
    pub fn new() -> Self {
        Self {
            client: reqwest::Client::new(),
        }
    }
}

impl Default for ArxivRssFeed {
    fn default() -> Self {
        Self::new()
    }
}
```

### 2. Implement the Trait

```rust
#[async_trait]
impl SubscriptionSource for ArxivRssFeed {
    fn source_type(&self) -> &str {
        "arxiv-rss"
    }

    fn display_name(&self) -> &str {
        "ArXiv RSS Feed"
    }

    fn description(&self) -> &str {
        "Subscribe to new papers from ArXiv categories via RSS"
    }

    fn default_config(&self) -> serde_json::Value {
        serde_json::json!({
            "categories": ["cs.AI", "cs.CL"],
            "max_results": 50
        })
    }

    async fn fetch(
        &self,
        config: &serde_json::Value,
        since: Option<chrono::DateTime<chrono::Utc>>,
    ) -> Result<Vec<SubscriptionItem>, SubscriptionError> {
        // 1. Read config
        let categories = config["categories"]
            .as_array()
            .map(|arr| arr.iter().filter_map(|v| v.as_str()).collect::<Vec<_>>())
            .unwrap_or_default();

        // 2. Build request URL
        let category_query = categories.join("+OR+");
        let url = format!(
            "http://export.arxiv.org/api/query?search_query=cat:{}&sortBy=submittedDate&sortOrder=descending&max_results=50",
            category_query
        );

        // 3. Fetch and parse the response
        let response = self.client
            .get(&url)
            .header("User-Agent", "Zoro/0.1.0")
            .send()
            .await?;

        let body = response.text().await?;

        // 4. Parse XML/Atom feed into SubscriptionItems
        //    (parse logic omitted for brevity)
        let items: Vec<SubscriptionItem> = parse_atom_feed(&body, since)?;

        Ok(items)
    }
}
```

### 3. Register the Module

In `crates/zoro-subscriptions/src/lib.rs`, add the module and re-export:

```rust
pub mod source;
pub mod huggingface;
pub mod arxiv_rss;        // Add this
pub mod error;

pub use source::SubscriptionSource;
pub use huggingface::HuggingFaceDailyPapers;
pub use arxiv_rss::ArxivRssFeed;  // Add this
pub use error::SubscriptionError;
```

### 4. Register in the Desktop App

In the subscription poller (or wherever sources are instantiated), add the new source to the registry. The pattern used is to match on `source_type` when dispatching `fetch()` calls:

```rust
// In the subscription poller
match subscription.source_type.as_str() {
    "huggingface-daily" => {
        let source = HuggingFaceDailyPapers::new();
        source.fetch(&config, since).await
    }
    "arxiv-rss" => {
        let source = ArxivRssFeed::new();
        source.fetch(&config, since).await
    }
    _ => Err(SubscriptionError::SourceNotFound(subscription.source_type.clone()))
}
```

### 5. (Optional) Add a Default Subscription

In `apps/desktop/src-tauri/src/lib.rs`, the `ensure_default_subscriptions` function creates default subscriptions on first launch:

```rust
fn ensure_default_subscriptions(db: &zoro_db::Database) {
    use zoro_db::queries::subscriptions;
    let subs = subscriptions::list_subscriptions(&db.conn).unwrap_or_default();

    if !subs.iter().any(|s| s.source_type == "huggingface-daily") {
        let _ = subscriptions::create_subscription(
            &db.conn,
            "huggingface-daily",
            "HuggingFace Daily Papers",
            Some(r#"{"auto_download_pdf": false}"#),
            60,
        );
    }

    // Add your default subscription here if desired
}
```

## Reference Implementation: HuggingFace Daily Papers

The `HuggingFaceDailyPapers` source in `crates/zoro-subscriptions/src/huggingface.rs` is the canonical reference implementation.

### Key Implementation Details

**API response types** -- Define Rust structs to deserialize the external API response:

```rust
#[derive(Debug, Deserialize)]
struct DailyPaperResponse {
    paper: HfPaper,
    title: Option<String>,
    #[serde(rename = "publishedAt")]
    published_at: Option<String>,
    upvotes: Option<i32>,
}

#[derive(Debug, Deserialize)]
struct HfPaper {
    id: String,          // ArXiv ID like "2603.05706"
    title: Option<String>,
    summary: Option<String>,
    authors: Option<Vec<HfAuthor>>,
}

#[derive(Debug, Deserialize)]
struct HfAuthor {
    name: Option<String>,
    #[serde(rename = "_id")]
    id: Option<String>,
}
```

**The fetch implementation** -- Fetches from the HuggingFace API and maps results to `SubscriptionItem`:

```rust
async fn fetch(
    &self,
    _config: &serde_json::Value,
    _since: Option<chrono::DateTime<chrono::Utc>>,
) -> Result<Vec<SubscriptionItem>, SubscriptionError> {
    info!("Fetching HuggingFace Daily Papers...");

    let url = "https://huggingface.co/api/daily_papers";
    let response = self.client
        .get(url)
        .header("User-Agent", "Zoro/0.1.0")
        .send()
        .await?;

    let papers: Vec<DailyPaperResponse> = response.json().await?;
    info!("Fetched {} papers from HuggingFace", papers.len());

    let items = papers.into_iter().map(|dp| {
        let arxiv_id = dp.paper.id.clone();
        let title = dp.paper.title.unwrap_or_else(|| dp.title.unwrap_or_default());
        let authors = dp.paper.authors
            .unwrap_or_default()
            .into_iter()
            .filter_map(|a| a.name.map(|name| Author {
                name,
                affiliation: None,
                orcid: None,
            }))
            .collect();

        let now = chrono::Utc::now().to_rfc3339();
        let id = uuid::Uuid::new_v4().to_string();

        SubscriptionItem {
            id,
            subscription_id: String::new(), // Set by the caller
            paper_id: None,
            external_id: arxiv_id.clone(),
            title,
            authors,
            abstract_text: dp.paper.summary,
            url: Some(format!("https://arxiv.org/abs/{}", arxiv_id)),
            pdf_url: Some(format!("https://arxiv.org/pdf/{}", arxiv_id)),
            html_url: Some(format!("https://arxiv.org/html/{}", arxiv_id)),
            upvotes: dp.upvotes,
            data: Some(serde_json::json!({
                "published_at": dp.published_at,
                "upvotes": dp.upvotes,
            })),
            fetched_date: now,
            added_to_library: false,
        }
    }).collect();

    Ok(items)
}
```

### Patterns to Follow

1. **Use `reqwest::Client`** stored as a field for connection reuse
2. **Set a `User-Agent` header** identifying Zoro
3. **Generate UUIDs** with `uuid::Uuid::new_v4().to_string()` for item IDs
4. **Leave `subscription_id` empty** -- the caller sets it
5. **Leave `paper_id` as `None`** -- it's set when the user adds the item to their library
6. **Set `added_to_library` to `false`** -- managed by the app
7. **Use `fetched_date` with `chrono::Utc::now().to_rfc3339()`**
8. **Store extra data** in the `data` field as JSON for source-specific information

## Config Schema Design Patterns

Config is stored as a `serde_json::Value` in the database, giving each source flexibility:

```json
// HuggingFace Daily Papers config
{
    "auto_download_pdf": false,
    "auto_download_html": false
}

// ArXiv RSS config (hypothetical)
{
    "categories": ["cs.AI", "cs.CL", "cs.LG"],
    "max_results": 50,
    "include_cross_listed": true
}

// Semantic Scholar config (hypothetical)
{
    "author_ids": ["1234567"],
    "keywords": ["transformer", "attention"],
    "min_citation_count": 10
}
```

Guidelines:
- Always provide sensible defaults via `default_config()`
- Keep the config schema flat and simple
- Use snake_case for JSON keys
- Document the config schema in your plugin's doc comments

## Fetch Lifecycle

```
1. Subscription poller triggers (every poll_interval_minutes)
       |
2. Read subscription from DB (source_type, config_json, last_polled)
       |
3. Look up the SubscriptionSource by source_type
       |
4. Call source.fetch(config, since=last_polled)
       |
5. For each returned SubscriptionItem:
   a. Check if external_id already exists in subscription_items
   b. If new, insert into subscription_items table
       |
6. Update subscription.last_polled timestamp
       |
7. Items appear in the UI feed view
```

## Testing Subscription Plugins

### Unit Tests

Test the parsing logic separately from network calls:

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_parse_response() {
        let json = r#"[{"paper": {"id": "2603.05706", "title": "Test Paper", ...}}]"#;
        // Test parsing logic
    }
}
```

### Integration Tests (Network)

Mark network-dependent tests with `#[ignore]` so they don't run in CI:

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[tokio::test]
    #[ignore] // Requires network access
    async fn test_fetch_daily_papers() {
        let source = HuggingFaceDailyPapers::new();
        let config = source.default_config();
        let items = source.fetch(&config, None).await.unwrap();
        assert!(!items.is_empty(), "Should fetch at least one paper");
        for item in &items {
            assert!(!item.title.is_empty(), "Paper should have a title");
            assert!(!item.external_id.is_empty(), "Paper should have an external ID");
        }
    }
}
```

Run ignored tests explicitly:

```bash
cargo test -p zoro-subscriptions -- --ignored
```
